[白鼠二]Hardy Alpha4 评测

Feb 4th, 2008 12:00 am | Comments

前文回顾：

————————————华丽的分割线————————————02.02

今天Alpha4放出来了。本来想刻盘装起来试试的，可惜手边没空盘。本着稳定压倒一切的宗旨，我还是继续用我的Gutsy算了。唉……等4月份吧 其实看到我上一个分割线并且了解我的人就会知道，我最终还是会忍不住去囧的。昨天用update-manager -d命令把系统更新了一遍。继续小白鼠生涯。

忍受完浙江电信限速的折磨后，终于成功升级。这次只报告了两个程序的崩溃，其中一个是opera。随后发现包括触摸屏驱动在内的很多Alpha3留下的BUG都修复了。这是个好现象。(02.11.重装Alpha4发现，触摸屏驱动并没有修复，只是因为我是从gutsy升级而来的，触摸屏驱动保留了而已)

简要发个汇报：

  • 各设备驱动工作正常（其实在Alpha3后期就已经正常了）。 除了无线指示灯仍旧不亮。
  • 开机进度条读入到1/8左右时停了很长时间，按Alt+Ctrl+F8看发现是在加载硬件驱动 。不知道和我的无线开着是否有关系（以前在用Feisty时候发现无线开着会在进度条1/5处停住）（02.11.重装发现，这个现象消失了。。。）
  • 开机关机音乐恢复了。在Gutsy里面消失了的关机音乐都恢复了。
  • 开启系统特效后系统仍旧很卡。而且开着特效很多页面仍旧出现一闪而过的花屏。
  • 待机终于恢复了，和Gutsy里面一样好用了。不过恢复的时候出现了这么一个东西——（囧，不能截图）——就是在登录对话框中，密码框的右边有个“USA”标签，点一下就变成一个问号。不知道干嘛用的。
  • 点一下右上角的日期 ，出现个“位置”按钮，点击就显示了世界地图。现在还是黑夜的地方居然还有阴影标记……
  • Audacious仍旧不能认识cue标签（今天早上发现Audacious可以升级到1.4.6了，高兴了一下下，结果升级完毕还是不认CUE）
  • Amarok仍旧不能正确读取cue时间戳（开始怀疑是自己机子的问题了。为什么yegle的就好用呢？）
  • 那个"Language and Location "又没了，匆匆登场还没表演就谢幕了……不知道是不是因为我没用光盘安装的缘故。
  • Opera本来美化好的字体现在又很难看了。。。。
  • 新增几个软件，Transmission, Avahi SSH/VNC server brower(这两个在Alpha3里面似乎就有了，不过用还是今天才用)，Barasero，按照TOY还有Yegle上介绍，我还装上了ufw，试了试它的傻瓜化操作。
  • (02.06)汇报一个比较郁闷的问题：以前很自豪Linux下不根据扩展名来判断文件打开方式，可是在Alpha4的Nautilus中居然根据扩展名来判断打开方式了，而且图标很多也不正常了。
  • (02.11)这个问题直接导致我又回到了gutsy：重新安装了alpha4后，发现nautilus出了问题。在terminal下执行nautilus命令，出现一个错误提示：Unable to add monitor: Not supported。看来是显示器或者显卡的问题。去系统里面看了一下，“显示器与显卡”中，居然没有显示器设备！到网上搜了一下，没有解决，懒得搜了。换回gutsy。
  • (02.21)CheckGmail的乱码问题自动解决了(还是怀疑和Perl版本有关系)
  • (02.21)升级的时候自动把我的Firefox2变成了Firefox3。FF3用倒是挺好用，就是插件不兼容。yegle告诉我一个Nightly Test Tool插件，可以跳过兼容性检查。感觉挺好用。
  • (02.21)字体突然变回了文泉驿宋体。还是在yegle的提示下，备份并删除了/etc/fonts/conf.avail/85-xfonts-wqy.conf这个文件，然后重新刷新字体缓存就可以了。
明天凌晨就该出Alpha5了，不过我也没空专门格盘重装评测了。以后dist-upgrade发现什么再写。
